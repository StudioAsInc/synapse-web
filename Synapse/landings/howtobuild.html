Backup Your Data

Critical: Avoid Data Loss

If you use the "Easy Start" zip, you must back up your existing projects first.

  1. Using a file manager, go to your Internal Storage and find the named .sketchware. Click here If you can't find the folder.
  2. Rename it to .sketchware_BACKUP. This saves everything.
2

Install Project Files

Choose your preferred method:

For Easy Start (.zip):

  1. Extract the downloaded Full_Project_Data.zip.
  2. Move the new .sketchware folder to your Internal Storage.

For Advanced Users:

  1. In Sketchware Pro, use "Restore Project" to import the .swb file.
  2. Extract the local_libs.zip and manually move its contents into the .sketchware/libs/local_libs directory.
3

Configure Firebase

  1. Go to the Firebase Console, create a project, and add an Android app with package name com.synapse.social.studioasinc
  2. Download the google-services.json and import it into the Sketchware project via Library Manager > Firebase.
  3. In Firebase, enable Authentication > Sign-in method > Email/Password.
  4. In Firebase, go to Realtime Database or Firestore, create a database, and navigate to the Rules tab. Paste the following rules to secure your data: 
    {
  "rules": {
    // Only authenticated users can read or write
    ".read": "auth != null",
    ".write": "auth != null",

    "users": {
      "$uid": {
        // A user can only write to their own user node
        ".write": "$uid === auth.uid"
      }
    },
    "posts": {
      "$postId": {
        // Anyone authenticated can read posts
        ".read": "auth != null",
        // Only the post owner can write/edit their post
        ".write": "!data.exists() || data.child('authorUid').val() === auth.uid"
      }
    }
  }
}
4

Build & Run

You're ready to compile! Tap the "Run" button in Sketchware Pro, then "Install" the generated APK. Congratulations, your client is live!

Troubleshooting & FAQ

Stuck? Check these common issues.

App fails to compile or shows a DEX error.

This usually means a library issue. Ensure you have installed the "Full Project Data" correctly, or manually placed the local libraries in the right folder. Also, try enabling/disabling AAPT2 in Sketchware's settings.

Login/Registration fails silently.

Check two things: 1) Did you import your own google-services.json correctly? 2) Did you enable the "Email/Password" sign-in provider in your Firebase project's Authentication section?

Data is not saving or loading from the database.

This is almost always a Firebase Rules issue. Go to your Realtime Database/Firestore rules and make sure they are not set to deny all reads/writes. Use the sample rules provided in Step 3 as a starting point.
